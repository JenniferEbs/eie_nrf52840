Git Cheatsheet

git commit  
Records changes to the repository.  
Arguments:  
  -m "message" adds a commit message inline  
  --amend modifies the previous commit  
Behavior changes with arguments: Yes. --amend edits the last commit instead of creating a new one.

git branch  
Lists, creates, or deletes branches.  
Arguments:  
  branch-name creates a new branch  
  -d branch-name deletes a branch  
  (no arguments) lists all branches  
Behavior changes with arguments: Yes. No arguments list branches; with one, it creates or deletes depending on flags.

git checkout  
Switches branches or restores files.  
Arguments:  
  branch-name switches to that branch  
  -b new-branch creates and switches to a new branch  
  file-name restores a file from a commit  
Behavior changes with arguments: Yes. It switches branches or restores files depending on what you pass.

git pull  
Fetches and merges changes from a remote repository.  
Arguments:  
  origin branch-name pulls from a specific remote and branch  
Behavior changes with arguments: Slightly. You can specify where to pull from.

git push  
Sends your commits to a remote repository.  
Arguments:  
  origin branch-name pushes to a specific remote and branch  
  --force forces the push  
  -u <remote> <branch> sets the upstream tracking reference  
Note about -u:  
  The -u option tells Git to remember the remote and branch you're pushing to.  
  After using it once, you can just type git push or git pull without specifying the remote and branch again.  
  It's like saying, "From now on, this branch should track that remote branch."  
Behavior changes with arguments: Yes. You can push different branches, force updates, or set upstream tracking. -u <remote> <branch> sets the upstream tracking reference  

git merge  
Combines changes from one branch into another.  
Arguments:  
  branch-name merges that branch into your current one  
Behavior changes with arguments: Yes. The branch you specify determines what gets merged.

git rebase  
Moves or replays commits from one branch onto another.  
Arguments:  
  branch-name rebases current branch onto the given one  
  --interactive or -i lets you edit, squash, or reorder commits  
Behavior changes with arguments: Yes. Interactive mode gives more control over commit history.

git fetch  
Downloads changes from a remote repository but doesnâ€™t merge them.  
Arguments:  
  origin fetches from a specific remote  
  --all fetches from all remotes  
Behavior changes with arguments: Slightly. You control what remote or branches to fetch.

git add  
Stages changes to be committed.  
Arguments:  
  file-name stages a specific file  
  . stages all changes in the current directory  
Behavior changes with arguments: Yes. You can stage individual files or everything at once.

git status  
Shows the current state of the working directory and staging area.  
Arguments:  
  (none) shows modified, staged, and untracked files  
Behavior changes with arguments: No. It always shows the current status.

git log  
Displays the commit history.  
Arguments:  
  --oneline shows a simplified version  
  --graph shows a visual representation of branches and merges  
Behavior changes with arguments: Yes. Different flags change how the history is displayed.
